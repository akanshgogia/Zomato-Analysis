# 🍽️ Zomato Restaurant Data Analysis

## 📌 Project Overview
This project involves exploratory data analysis (EDA) on a Zomato restaurant dataset to uncover insights into dining options, cost distributions, customer ratings, and service availability. 

## 🛠️ Data Preprocessing & Quality
* **Dataset Shape:** The dataset contains 148 restaurant listings and 7 core features. 
* **Data Cleansing:** Numeric ratings were extracted from text strings, and any missing values were imputed using the dataset's median score. 
* **Data Quality:** The cleaned dataset features high data integrity, with zero duplicate rows and no missing values across its columns.

## 📊 Key Findings & Insights

### 💰 Cost & Pricing
* **Price Range:** The approximate cost for two people ranges from a budget-friendly 100 to a premium 950.
* **Average Spending:** The mean cost for a meal for two is approximately 418.

### ⭐ Customer Ratings
* **Rating Spectrum:** Customer ratings span from a low of 2.6 up to an impressive 4.6 out of 5.
* **Average Rating:** The overall average rating sits at 3.63, indicating a generally positive dining landscape within the dataset.

### 🏪 Restaurant Categories
* **Dining Dominance:** "Dining" is by far the most prominent establishment type, making up 110 of the 148 listings.
* **Other Formats:** "Cafes" make up the second-largest category with 23 listings, while "Buffets" (7 listings) and "Other" types (8 listings) represent smaller niche markets.

### 🛵 Service Availability (Online Ordering & Table Booking)
* **Online Orders:** A significant majority of the restaurants (90 listings) operate strictly offline, compared to the 58 establishments that accept online orders.
* **Table Reservations:** Table booking is a highly exclusive feature, with 140 restaurants choosing not to offer reservations and only 8 actively providing the service.
